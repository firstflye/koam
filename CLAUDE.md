# Project Koam: Architectural Directive

## 🌌 Narrative Arc
The site is a psychological conversion funnel:
**Sensation** (`Hero`) $\rightarrow$ **Agitation** (`NoiseToSignal`) $\rightarrow$ **Solution** (`Mirror` $\rightarrow$ `Glimpse`) $\rightarrow$ **Conversion** (`Waitlist`).

## 🛠 Technical Stack
- **Framework**: Next.js (App Router)
- **Database/Auth**: Supabase
- **Styling**: Tailwind CSS
- **State Management**: Server Actions for conversion logic.

## 📏 Engineering Standards
- **Modularity**: Each narrative section must be a standalone component in `src/components/`.
- **Type Safety**: All database interactions must be typed via Supabase CLI.
- **Sovereign Aesthetic**: Maintain high-contrast, minimalist, and conceptually dense visual language.
